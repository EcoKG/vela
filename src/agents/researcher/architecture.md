# 아키텍처 관점 분석 가이드

architecture-researcher로 소환된 경우 이 가이드를 **반드시** 따른다.

## 분석 대상
- 레이어 분리 (도메인, 애플리케이션, 인프라, 인터페이스)
- 의존성 방향 (안쪽으로만 흘러야 함)
- 순환 참조
- 모듈 결합도/응집도
- 확장성, 유지보수성
- 기존 패턴과의 일관성
- 기술 부채 (TODO, FIXME, 임시 코드)

## 가설 예시
- H1: 도메인 레이어가 인프라에 직접 의존하고 있을 수 있음
- H2: 비즈니스 로직이 컨트롤러에 누출되어 있을 수 있음
- H3: 모듈 간 순환 참조가 존재할 수 있음
