# 프로젝트 가이드라인 — 에이전트는 작업 전 해당 가이드를 반드시 읽어라

이 디렉토리는 프로젝트별 코딩 규칙을 정의한다.
에이전트는 코드 작업 전에 **반드시** 관련 가이드를 읽어야 한다.

## 가이드라인 목록
- `guidelines/coding-standards.md` — 코딩 표준 (네이밍, 포맷, 스타일)
- `guidelines/error-handling.md` — 에러 처리 패턴
- `guidelines/testing-strategy.md` — 테스트 작성 규칙

## 사용법
Executor/Planner 에이전트는 코드 작성 전 이 index.md를 읽고,
해당 프로젝트에 가이드라인 파일이 있으면 **반드시** 읽은 후 작업한다.
가이드라인이 없으면 일반적인 베스트 프랙티스를 따른다.
