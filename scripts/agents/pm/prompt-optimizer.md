# 프롬프트 최적화 — 반드시 실행

이 절차는 모든 모드(Explore/Research/Develop)에서 **예외 없이 최우선 실행**한다.

## 흐름
1. 사용자 요청의 대상/범위/목적/기술적 맥락을 분석한다
2. 반드시 AskUserQuestion으로 확인한다 — 충분하면 "이대로 진행"을 Recommended로 표시
3. "이대로 진행" → 원본 프롬프트로 scale 선택
4. 보완 선택 → 세부 수집 → PM이 조립 → 사용자 확인 → scale 선택

## 보완 항목
- **대상 파일/모듈 지정**: "어떤 파일/모듈을 수정할까요?" → Other로 경로 입력
- **범위 좁히기**: 특정 클래스 / 함수 / 레이어 / Other
- **문제 상세**: 버그 재현 조건 / 에러 메시지 / 기대 동작 / 영향 범위

복수 보완 가능 — 하나 수집 후 추가 필요하면 다시 표시한다.

## 조립 규칙
1. **대상**: 어떤 파일/모듈/클래스를 수정하는가
2. **작업**: 무엇을 하는가
3. **상세**: 재현 조건, 에러, 기대 동작
4. **범위**: 수정 범위 한정

조립 후 반드시 텍스트로 출력하고 AskUserQuestion으로 확인받는다.
승인된 프롬프트가 `vela-engine init`의 request가 된다.
